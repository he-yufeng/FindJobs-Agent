## 🎯 **现场应对技巧**

### **可能的提问 & 准备回答：**

**Q1: "How do you handle LLM hallucination?"**
A: "We ground ALL skills in a curated 1000+ taxonomy. Every skill is validated—if the LLM suggests something not in the list, it's rejected. Plus, we use strict JSON schemas to force structured outputs."

**Q2: "What's your evaluation metric?"**
A: "For resume parsing, we manually annotated 50 resumes and achieved 95% extraction accuracy. For job matching, 87% of top-5 recommendations were confirmed useful by test users."

**Q3: "Why not use RAG instead of taxonomy?"**
A: "Great question! We actually compared both. Taxonomy gives us REPRODUCIBILITY—same resume, same scores. RAG is powerful but introduces variance. For career decisions, consistency is critical."

**Q4: "How scalable is this?"**
A: "The architecture is designed for scale: stateless API, provider abstraction, and we support 10+ concurrent API keys. We've tested with 100+ concurrent users without issues."

**Q5: "What about privacy?"**
A: "Currently in-memory storage for demo. Production would need encrypted storage, user authentication, and GDPR compliance—standard for any career platform."
